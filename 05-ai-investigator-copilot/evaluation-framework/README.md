# Evaluation Framework

Measuring Quality, Accuracy and Trust in AI-Generated Financial Crime Intelligence

---

# Executive Summary

The AI Investigator Copilot produces investigator-facing intelligence products including alert rationales, investigator summaries, evidence packs, intelligence reports and case packages.

To support operational use, all outputs must be accurate, explainable, traceable and useful.

The Evaluation Framework defines how AI-generated outputs are assessed, monitored and continuously improved.

The objective is not simply to measure whether the AI generates text.

The objective is to measure whether the AI produces high-quality investigative intelligence.

---

# Evaluation Objectives

The framework evaluates AI performance across five dimensions:

1. Accuracy
2. Explainability
3. Completeness
4. Consistency
5. Investigator Value

Each output should satisfy all five dimensions before operational deployment.

---

# Evaluation Principles

The framework follows several core principles:

- Evidence-based outputs
- Traceable reasoning
- Human review
- Repeatable testing
- Continuous improvement
- Regulatory defensibility

AI-generated intelligence should always be explainable and auditable.

# Evaluation Categories

## Accuracy

The output correctly reflects the underlying evidence.

Questions include:

- Are facts correct?
- Are relationships correctly described?
- Are risks accurately represented?
- Are conclusions supported?

---

## Explainability

The output clearly explains reasoning.

Questions include:

- Can an investigator understand the conclusion?
- Is the reasoning transparent?
- Are assumptions identified?
- Is supporting evidence referenced?

---

## Completeness

The output contains all relevant information.

Questions include:

- Are key entities included?
- Are key transactions included?
- Are significant risks included?
- Are critical observations included?

---

## Consistency

The output remains stable across repeated executions.

Questions include:

- Are conclusions repeatable?
- Are explanations consistent?
- Is risk interpretation stable?

---

## Investigator Value

The output improves investigative effectiveness.

Questions include:

- Does it save time?
- Does it improve understanding?
- Does it improve decision quality?
- Does it reduce manual effort?

---

# Hallucination Testing

A critical evaluation objective is the identification of hallucinations.

Hallucinations occur when an AI system generates information not supported by available evidence.

Examples include:

- Invented entities
- Invented transactions
- Unsupported conclusions
- False relationships
- Fabricated intelligence

All outputs should be validated against source evidence.

---

# Grounding Validation

Grounding validation ensures outputs remain anchored to source material.

```text
Source Evidence
        ↓
AI Processing
        ↓
Generated Output
        ↓
Evidence Validation
```

Every significant conclusion should be traceable to supporting evidence.

---

# Traceability Assessment

Outputs should support full auditability.

Questions include:

- Which evidence supported the conclusion?
- Which analytics contributed?
- Which typologies contributed?
- Which intelligence sources contributed?
- Which prompts were used?

All outputs should be explainable after generation.

---

# Human Evaluation Framework

Human reviewers assess generated outputs using standard scoring criteria.

Example scoring dimensions:

| Category | Score |
|-----------|---------|
| Accuracy | 1-5 |
| Completeness | 1-5 |
| Explainability | 1-5 |
| Usefulness | 1-5 |
| Overall Quality | 1-5 |

Review feedback supports continuous improvement.
